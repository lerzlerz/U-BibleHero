# 《心灵迷宫：圣所探秘》游戏设计方案 (Game Design Document)

**文档版本**：v1.1.0  
**完结日期**：2026年7月2日  
**设计定位**：一款融合 DISC 性格测评的 2D 顶视角迷宫探索与圣经叙事 RPG 游戏。

本设计方案是由 `grill-me` 设计评审机制深度探讨后沉淀达成的共识，旨在为 Unity 引擎开发提供最终的技术和策划设计规范。

---

## 一、 游戏核心循环 (Core Loop)

```
[玩家出生] ➡️ [在圣所之林漫游探索] 
                 │
                 ├── 遭遇20位心灵导引者 (触发动态题库，最少维度补偿法抽题)
                 └── 遭遇物理关卡 (如断桥、警告石碑，静默记录操作行为)
                 │
[答满32题 (80% 进度)] ➡️ [开启圣殿门] ➡️ [圣殿觉醒仪式] 
                                        │
[开启第二阶段叙事冒险] ⬅️ [解锁专属圣徒职业 & 解锁相应角色故事线]
```

---

## 二、 系统架构与子模块设计

### 1. 动态题目池系统 (Dynamic Question Pool System)
*   **规则**：森林为开放世界，共部署 20 个 NPC。为防玩家疲劳，玩家只需完成 **80% 进度（答满 16 个 NPC，共 32 题）** 即可解锁大门通关。
*   **平衡算法：最少维度补偿法**
    *   在抽取题目时，系统实时扫描已答题中 D, I, S, C 维度的占比。
    *   当玩家与任何一个 NPC 交互时，系统会优先在**当前答题数最少**的那个维度中抽取一道未答题。
    *   *效果*：完美解决了自由路线下的“机会偏斜”问题，保证在 80% 进度截断时，各维度题数仍呈 1:1:1:1 绝对平衡。

### 2. 游戏二阶段：圣徒觉醒与故事探索 (Awakening & Narrative RPG)
测试结果不只是一个静态的分数报告卡，而是作为游戏的**“角色创建/转职系统”**：
*   **职业解锁**：测试终点，系统根据 DISC 矩阵匹配 16 种圣徒组合之一（如：D/C 匹配保罗，I/C 匹配大卫），为玩家解锁对应的**“圣徒职业”**（各职业拥有独特的属灵技能与数值面板）。
*   **故事探索**：通关后开启游戏第二阶段——**“圣卷编年史（Chronicle Quests）”**。玩家将使用解锁 of 职业角色，亲身探索和经历该圣经人物的经典史诗故事（如：大卫战歌利亚、保罗的宣教旅程、以斯帖救同胞等），在叙事任务中实践其“盲点突破”的成长课题。

## 三、 Unity 核心数据模型 (C# Scripts)

### 1. 动态题目池管理器 (DynamicQuestionPool.cs)
```csharp
// DynamicQuestionPool.cs
using System.Collections.Generic;
using UnityEngine;

public class DynamicQuestionPool : MonoBehaviour
{
    public static DynamicQuestionPool Instance;

    public List<QuestionData> allQuestions; // 全局40道题库 (D, I, S, C 各10道)
    private Dictionary<int, string> playerAnswers = new Dictionary<int, string>();

    void Awake()
    {
        Instance = this;
    }

    /// <summary>
    /// 最少维度补偿抽题算法
    /// </summary>
    public QuestionData GetNextBalancedQuestion()
    {
        // 1. 统计当前已回答的 D, I, S, C 数量
        Dictionary<string, int> counts = new Dictionary<string, int>()
        {
            { "D", 0 }, { "I", 0 }, { "S", 0 }, { "C", 0 }
        };

        foreach (var ans in playerAnswers.Values)
        {
            if (counts.ContainsKey(ans)) counts[ans]++;
        }

        // 2. 找出当前回答次数最少的维度
        string leastDimension = "D";
        int minCount = int.MaxValue;
        foreach (var kvp in counts)
        {
            if (kvp.Value < minCount)
            {
                minCount = kvp.Value;
                leastDimension = kvp.Key;
            }
        }

        // 3. 从题库中筛选该维度下未回答的题目
        List<QuestionData> candidateQuestions = allQuestions.FindAll(q => 
            !playerAnswers.ContainsKey(q.id) && q.GetDimensionType() == leastDimension
        );

        // 如果该维度题目已经全部答完，退化为随机找未答题目
        if (candidateQuestions.Count == 0)
        {
            candidateQuestions = allQuestions.FindAll(q => !playerAnswers.ContainsKey(q.id));
        }

        if (candidateQuestions.Count == 0) return null; // 全部答完

        // 4. 随机抽出一道
        return candidateQuestions[Random.Range(0, candidateQuestions.Count)];
    }

    public void RecordAnswer(int questionId, string type)
    {
        playerAnswers[questionId] = type;
    }
}
```
